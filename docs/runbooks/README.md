# Runbooks

## Purpose

This folder contains operational procedures for recovering, debugging, and safely operating the platform.

Runbooks should be written as instructions that can be followed during a real incident or maintenance task.

## Does Not Belong Here

Architecture decisions, general course notes, Terraform source code, Kubernetes manifests, and secrets do not belong here.

## Ownership

Runbooks are documentation. They describe how operators should act when the platform needs recovery, verification, or controlled change.
